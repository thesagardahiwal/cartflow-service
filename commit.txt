docs: update swagger, postman collection and automation scripts

This commit brings the documentation and docker orchestration in line with the new architecture.

- Updated `swagger.json` with comprehensive OpenAPI 3.0 specs for public endpoints, including Idempotency headers, BearerAuth security schemes, and proper payload formatting.
- Fixed Swagger UI CORS/Host routing issues by utilizing relative `servers.url` configuration.
- Completely rewrote `postman_collection.json` to include the newest endpoints (Analytics, Auth, Cart, Checkout) with embedded idempotency keys and proper HTTP methods.
- Converted the Checkout endpoint route from `GET` to `POST` to conform to REST conventions for mutating states, and successfully bound it to the Idempotency middleware.
- Moved `seed.ts` inside the `src` directory so it natively compiles with `tsc` during Docker build steps.
- Chained `seed.js` into the `package.json` startup script so the Mongo cluster seeds automatically when containers spin up.
